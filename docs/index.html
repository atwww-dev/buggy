Buggy Documentation

A lightweight, embeddable bug reporting tool for web applications

Introduction

Buggy is a simple yet powerful bug reporting tool that can be easily integrated into any web application. It allows users to capture screenshots, annotate them, and submit detailed bug reports directly to your Trello board.

Features

  • One-click screenshot capture
  • Advanced annotation tools (pen, arrow, rectangle, circle, text)
  • Color selection for annotations
  • Automatic capture of browser and URL information
  • Customizable form fields
  • Direct integration with Trello
  • Responsive design
  • Easy to integrate

Installation

Client-side Setup

Add the following to your HTML file:

<script src="https://cdn.jsdelivr.net/npm/html2canvas@1.4.1/dist/html2canvas.min.js"></script>
<script src="path/to/buggy.js"></script>
<script>
  new Buggy({
    apiUrl: 'http://your-server:3000/api/feedback',
    buttonText: 'Report Bug',
    buttonPosition: 'bottom-right'
  });
</script>

Server-side Setup

1. Install dependencies:

cd server
npm install

2. Configure environment variables in .env:

PORT=3000
TRELLO_API_KEY=your_api_key_here
TRELLO_TOKEN=your_token_here
TRELLO_LIST_ID=your_list_id_here

3. Start the server:

npm start

Configuration Options

Available Options

{
  apiUrl: string,          // Your server endpoint
  buttonText: string,      // Text for the report button
  buttonPosition: string,  // Position of the button (top-left, top-right, bottom-left, bottom-right)
  categories: string[],    // Custom categories for bug classification
  priorities: string[],    // Custom priority levels
  formFields: {           // Custom form fields
    title: boolean,
    description: boolean,
    category: boolean,
    priority: boolean,
    steps: boolean
  }
}

Annotation Tools

  • Pen (✏️): Freehand drawing tool
  • Arrow (➡️): Draw arrows to point to specific areas
  • Rectangle (⬜): Draw rectangles to highlight areas
  • Circle (⭕): Draw circles to emphasize elements
  • Text (T): Add text annotations

All annotations can be drawn in different colors using the color picker tool.

Trello Integration

Buggy automatically creates Trello cards with:

  • Screenshot with annotations
  • Bug title and description
  • Category and priority
  • Reproduction steps
  • Technical details (URL, browser, timestamp)

Make sure to keep your Trello API credentials secure and never expose them in client-side code.

Browser Support

Buggy works in all modern browsers that support:

  • HTML5 Canvas
  • CSS3
  • ES6+ JavaScript

Troubleshooting

Common Issues

  • Screenshot not capturing: Ensure html2canvas is properly loaded
  • Annotations not saving: Check canvas size and permissions
  • Trello integration failing: Verify API credentials and list ID
